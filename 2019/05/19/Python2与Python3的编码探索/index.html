Introduction:
由于Python课老师布置了一个题目，做完之后前半学期不用去上课，我选了一个关于天气爬虫的项目，耗时大概一周，磕磕绊绊的才发现自己对Python的掌握还是不够熟练，基于上述遇到的一个比较头疼的问题就是Python2关于字符编码的问题，Python2和Python3对于字符的处理机制是不一样的，固闲暇时间静下心来处理一下编码这个版块的问题

编码

发展

8bit=1byte 1byte->255 2byte->65535 4byte->4294967295

初期： ASCII 127个字符编码
中期： Unicode
后期： UTF-8

因为汉字不能用1个字节编码，还有其他各种语言，于是出现了统一的编码Unicode,可是Unicode占2个字节存储空间扩大一倍，固出现了UTF-8的可变长编码，英文1byte,汉字3byte。

ASCII->1byte Unicode->2byte (ASCII码前面补0)

编码的意思是将Unicode字符按照编码规则（如UTF-8）编成字节序列

Python3一些存储特性

  1. 计算机内存 Unicode编码，保存硬盘或传输转换为UTF-8编码。
  2. ord()整数表示;chr()编码转化字符。
  3. 字符串类型为str,保存到磁盘str转化为bytes
  4. 英文str用ASCII encode()为bytes,含中文str只能用UTF-8编码为bytes;否则会报UnicodeEncodeError
  5. 读取磁盘或网络为bytes流，需要decode(),bytes中无法显示的ASCII字符的字节用\x##表示
  6. 一小部分错误字节可传入参数errors='ignore'忽略。
  7. len()计算str字符数，计算bytes的字节数。
  8. 1中文经utf-8编码占3个字节，英文1个字节

Python2和Python3共性与区别

Python3 Python2
str——-unicode 字符串
bytes——str 字节串

共性

在Windows系统中需要使用gbk来解码， 无论utf-8还是gbk都只是一种编码编码规则，一种把Unicode数据编码成字节数据的规则。

不同

  • python3中将utf-8或者gbk等编码的字节数据解码为Python3中的str类型， utf-8编码的bytes <—> str
  • python2中将 utf-8或者gbk等编码的str解码为Python2中的Unicode， utf-8编码的str <—> unicode

Python2编码

str类型和Unicode类型，分别存储的是字节数据和Unicode数据

Python2 str(s)和unicode(s)

str(s)和unicode(s)是两个工厂方法，分别返回str字符串对象和unicode字符串对象，str(s)等价s.encode(‘ascii’)
unicode(s)等价于s.decode(‘ascii’)

固解析如u’你好’这类的unicode编码需指定正确编码如GBK或ASCII

Python2 乱码本质原因

所有出现乱码的原因都可以归结为字符经过不同编码解码在编码的过程中使用的编码格式不一致

对于变量内容本身为unicode形式的字符串：
s = 'id\u003d215903184\u0026index\u003d0\u0026st\u003d52\u0026sid’

s.decode('unicode-escape')

Reason:

Solution

python编码

python2.x从外部获取的内容都是string编码，其内部分为String编码与Unicode编码，而String编码又分为UTF-8，GBK，GB2312等等。因此为了避免不同编码造成的报错，python内部最好都转化为unicode编码，在输出时再转化为str编码 。可以用encode()/decode()函数，将string与unicode编码互换。

源代码编码

源代码编码指的是python程序本身的编码，默认为ascii,会隐式编码encode(‘ascii’)。

文件编码

python程序从文件中获取的内容的编码格式,sys.getfilesystemencoding()得到。

Error

输出无异常，与Python程序结合可能会造成异常.

Solution

从文件中读取的编码格式为Str，GBK格式;可使用codecs转化为unicode
f=codecs.open("test.txt", encoding='gbk').read()
or
open(“”,”r”).read().decode(“gbk”)

Python2编码tips

tip1

  • python代码内部请全部使用unicode编码，在获取外部内容时，先decode为unicode，向外输出时再encode为str

  • 在定义变量或者正则时，也定义unicode字符，如a=u”中文”；res=r””+u”正则”。
